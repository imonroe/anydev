Do the following:
First, reflect on what you have learned in this session, and make any documentation updates necessary to remember the lessons in the future.
Next, create a new branch, commit your changes, push to github and open a pull request.
Then, watch the pull request on Github and ensure that tests and other github actions all succeed. If they do not, examine any test failures. Investigate and correct them. Always lint your changes. Push the corrected code to the branch in the PR, and then examine the test run to ensure the tests all pass. If they do not, continue this pattern until they do.
Finally, once the test are passing on Github, check out the `main` branch, and ask what to do next.
